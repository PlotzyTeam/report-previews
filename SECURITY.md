# Security Policy

## Reporting a Vulnerability

If you believe you've found a security vulnerability in this repository, please **do not open a public issue**. Report it privately through GitHub's [private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability):

1. Go to the **Security** tab of this repository
2. Click **Report a vulnerability**
3. Fill in the details

## What to Include

- A description of the issue and its potential impact
- Steps to reproduce (proof-of-concept welcome)
- Affected version, commit, or branch
- Any relevant logs, screenshots, or environment details

## Response Timeline

- **Acknowledgement:** within 72 hours
- **Triage & severity assessment:** within 5 business days
- **Fix or mitigation plan:** communicated after triage

## Disclosure Policy

We follow coordinated disclosure. Please give us a reasonable window to investigate and remediate before any public disclosure. We will credit reporters in the advisory unless anonymity is requested.

## Scope

This policy applies to the latest code on the default branch. Older versions are not actively supported unless explicitly stated.
